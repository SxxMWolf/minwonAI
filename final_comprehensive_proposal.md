# 차세대 대국민 AI 행정 서비스 통합 제안서
## : 수요자 중심 대화형 단일창구 및 고신뢰 데이터 거버넌스 구현

---

### Ⅰ. 문제 정의 및 배경

#### 1.1 행정 인프라의 구조적 한계: '역전된 UX'와 데이터 사일로
- **기관 중심의 파편화**: 현재 민원 시스템은 국민이 행정 구조(기관, 부서)를 이해해야만 서비스를 이용할 수 있는 **역전된 UX** 구조임.
- **데이터 단절**: 부처별 개별 DB 운영으로 인해 동일한 정보가 중복 요구되며, 민원 간 관계(선후/병렬) 정의 부재로 복합 민원의 자동 처리가 원천적으로 불가능함.

#### 1.2 페르소나 및 상황 설정
- **대상**: 30대 직장인 김민원 씨 (서울에서 대전으로 이직 및 이사 예정)
- **상황**: 
    1. 전입신고뿐 아니라 자동차 주소 변경, 자녀 초등학교 전학, 지역별 청년 지원금 확인 등이 동시에 필요함.
    2. 김민원 씨는 각기 다른 4개 기관의 웹사이트를 방문하여 서류를 중복 업로드하고, 처리 상태를 개별적으로 확인해야 하는 불편을 겪음.
- **Goal**: "이사했어요"라는 단 한 마디로 AI가 모든 행정 절차를 설계하고 실행해 주는 환경 구축.

### Ⅱ. 서비스 시나리오 및 AI 역할 정의

#### 2.1 서비스 단계별 시나리오
1. **의도 인식 (Intent)**: 사용자가 음성/텍스트로 발화(예: "이번 달에 대전으로 이사 가요")하면 AI가 생활 이벤트를 포착.
2. **복합 태스크 분해 (Decomposition)**: 지식 그래프를 탐색하여 전입신고, 차량 변경 등 연관 민원을 자동 추출.
3. **실행 계획 수립 (Planning)**: 기관별 선후 관계를 고려한 실행 경로(DAG) 생성 및 사용자 승인.
4. **자율 실행 및 모니터링 (Execution)**: API 연동을 통해 민원 직접 대행 및 결과 실시간 HUD 안내.

#### 2.2 AI 단계별 역할 정의
- **Master Strategic Planner (플래너)**: 복합 민원의 맥락 분석 및 최적의 Action Plan 설계.
- **Specialized Execution Swarm (엑시큐터)**: 부처별 API 및 MCP 툴을 활용한 실제 행정 서비스 수행.
- **Execution Governor (오디터)**: 법률/규정 준수 여부 실시간 검증 및 리스크 점수 산출.

### Ⅲ. 기술 아키텍처 제안

| 계층 (Layer) | 주요 구성 요소 | 기술적 특징 |
| :--- | :--- | :--- |
| **Presentation** | HUD Navigator, 멀티모달 인터페이스 | React, STT/TTS, 대화창 내 실행형 UI (Summary Card) |
| **Application** | LangGraph Orchestrator, Temporal | 유향 비순환 그래프(DAG) 기반 워크플로우 제어 및 상태 유지 |
| **AI Layer** | BERT+LLM Pipeline, sLLM, Milvus | 2단계 의도 분류, On-prem 기반 고신뢰 RAG 및 XAI 구현 |
| **Data Layer** | **Knowledge Graph (Neo4j)**, Schema Registry | 민원 간 관계 모델링, OpenAPI 3.0 표준 스키마 및 매핑 |
| **Security & Gov** | **Hyperledger Fabric**, MS Presidio, MLflow | 블록체인 기반 불변 감사 로그, 실시간 개인정보 비식별화 |

### Ⅳ. 개발범위 및 단계별 구현 계획

#### 4.1 개발범위 정의
- **Core Engine**: 복합 민원 분해 엔진, 지식 그래프 모델링, 리스크 기반 HITL 시스템.
- **Domain Expansion**: 이사, 창업, 취업, 졸업, 실업 등 5대 생애 주기 핵심 도메인.
- **Common Infra**: 범정부 표준 API Gateway, 통합 행정 데이터 모델 및 보안 Safe Zone.

#### 4.2 단계별 구축 방안
1. **Phase 1 (인프라 구축)**: 통합 스키마 정의 및 고빈도 민원 시범 서비스 구축 (4개월).
2. **Phase 2 (연계 고도화)**: 부처 간 멀티에이전트 오케스트레이션 및 지식 그래프 확장 (8개월).
3. **Phase 3 (완전 전환)**: 전국 단위 데이터 거버넌스 확립 및 Proactive 행정 서비스 상용화 (12개월).

#### 4.3 주요 리스크 및 대응 방안
- **Hallucination (환각)**: XAI(Source Linking)와 이중 모델 검증(Cross-validation)을 통한 결정론적 응답 강제.
- **Data Integration (연계 저항)**: MCP(Model Context Protocol) 툴 표준화로 부처별 시스템 영향도 최소화 및 표준 스키마 준수 의무화.
- **Privacy (개인정보)**: 온프레미스 기반 sLLM 운영 및 물리적 분리된 Safe Zone 구축으로 외부 유출 원천 차단.

#### 4.4 프로토타입 구축 계획 (예시)
- **일정**: 착수 후 4개월 (분석 1, 개발 2, 검증 1)
- **예상 개발 비용**: **약 5.2억 원** (인프라 1.5억, 인건비 3억, 기타 0.7억 수준)

### Ⅴ. 기대효과 및 성과 지표

#### 5.1 가치 창출
- **국민**: 행정 탐색 비용 90% 절감, 복합 민원 처리 시간 50% 이상 단축.
- **행정**: 부처 간 칸막이 해소로 중복 업무 감소, 데이터 기반 정밀 정책 수립 가능.
- **사회**: 디지털 취약계층의 접근성 비약적 상승 및 AI 행정 신뢰도 제고.

#### 5.2 성과 측정지표 (KPI)
- **Time to Result**: 기존 대비 복합 민원 완료 소요 시간 50% 단축.
- **AI Task Accuracy**: AI의 자동 분해 및 실행 정확도 95% 이상 달성.
- **Citizen Satisfaction**: 서비스 사용 전후 행정 만족도 점수 30% 이상 향상.
- **Governance Zero-Gap**: AI 오판에 따른 법적 분쟁 및 개인정보 보안 사고 0건 유지.
